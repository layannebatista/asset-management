LEGENDA
  Tipo        : E2E (Cucumber BDD), UNITARIO, INTEGRACAO, CONTRATO, PERFORMANCE, SEGURANCA
  Criticidade : BLOCKER > CRITICAL > NORMAL > LOW
  Status      : AUTOMATIZADO | PENDENTE | NAO_APLICAVEL
  Camada      : indica o arquivo ou ferramenta onde o teste vive (ou deve viver)

NOTAS GERAIS
  - Todos os cenários E2E usam Spring MockMvc + profile bdd
  - Manutenção é obrigatória para ativos com status IN_MAINTENANCE
  - Transições de status: PENDING → INITIATED → COMPLETED/CANCELLED
  - Apenas ADMIN e GESTOR podem criar/aprovar manutenção
  - Descrição de manutenção: mínimo 10 caracteres, máximo 500 caracteres
  - GESTOR só pode criar manutenção para ativos da própria unidade
  - Conclusão restaura ativo para ASSIGNED se havia usuário atribuído antes (c.f. MS08 na camada de serviço)
