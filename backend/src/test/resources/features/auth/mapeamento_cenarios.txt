LEGENDA
  Tipo        : E2E (Cucumber BDD), UNITARIO, INTEGRACAO, CONTRATO, PERFORMANCE, SEGURANCA
  Criticidade : BLOCKER > CRITICAL > NORMAL > LOW
  Status      : AUTOMATIZADO | PENDENTE | NAO_APLICAVEL
  Camada      : indica o arquivo ou ferramenta onde o teste vive (ou deve viver)

NOTAS GERAIS
  - Todos os cenários E2E usam Spring MockMvc + profile bdd
  - Auth possui 4 endpoints principais: /auth/login, /auth/mfa/verify, /auth/refresh, /auth/logout
  - MFA é obrigatório somente para usuários com phoneNumber preenchido
  - Refresh token é rotacionado no uso e logout revoga todos os refresh tokens do usuário
  - Rate limit atual: login 10/min, mfa/verify 5/min, refresh 30/min
